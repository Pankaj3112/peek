# Peek v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build peek v1 — a CLI wrapper that captures dev server logs to disk and exposes them to Claude via a stdio MCP server with three tools (`list_sessions`, `get_logs`, `search_logs`).

**Architecture:** Single Go binary with five subcommands (`-- <cmd>`, `mcp`, `list`, `logs <id>`, `--version/--help`). The wrap form spawns the child under a pseudo-terminal, tees raw bytes to the user's terminal and a normalized stream to disk. The MCP server reads the on-disk session store (no shared in-memory state). Plugin manifests for Claude Code and Codex install via marketplace.

**Tech Stack:** Go 1.22+. Direct dependencies: `github.com/aymanbagabas/go-pty` (cross-platform pty), `github.com/oklog/ulid/v2` (session IDs), `golang.org/x/term` (raw mode). Standard library for everything else: `encoding/json`, `regexp` (RE2), `os/exec`, `os/signal`, `path/filepath`. Testing with stdlib `testing` package only — no testify.

**Source spec:** `docs/superpowers/specs/2026-05-03-peek-design.md`. Every task references the relevant spec section by heading; the executing session must read the spec for full behavioral detail before implementing each task.

**Convention for this plan:** code samples are intentionally omitted — the spec defines the data structures, response shapes, and algorithms precisely. Each task names the behavior, points at the spec section, and lists what the test must verify. The executing session writes the actual Go.

### Task 6: Session create + finalize

**Files:**
- Create: `internal/store/session.go`, `internal/store/session_test.go`

- [ ] **Step 1: Write failing test `TestSessionCreateAndFinalize`**

Test (table-driven, uses `t.TempDir()` and overrides `SessionsRoot` via a test-only injection point or `XDG_DATA_HOME`-style env var to redirect to the temp dir — pick whichever is cleaner):

- `Create(cwd, cmd)` returns a session handle. Side effects on disk: `<sessions-root>/<ulid>/` exists, `meta.json` inside contains a valid Meta with `status: "running"`, the wrapper's PID, the passed cwd and cmd, and a recent `started_at`.
- `Finalize(handle, exitCode)` updates `meta.json` to `status: "exited"`, fills `exited_at` (now), and `exit_code` (passed value).
- Reading the directory back via Task 7's `ReadMeta` produces the same struct just written.
- Two simultaneous `Create` calls (use a goroutine) produce two distinct directories.

- [ ] **Step 2: Run test, expect FAIL**

- [ ] **Step 3: Implement `Create` and `Finalize`**

`Create(cwd string, cmd []string) (*Session, error)`:
1. Generate ULID (Task 4).
2. Resolve `SessionDir(id)` (Task 5).
3. `os.MkdirAll(dir, 0o755)`.
4. Build a running Meta (Task 2's constructor) using `os.Getpid()` for the wrapper PID.
5. Write `meta.json` atomically: write to `meta.json.tmp`, fsync, rename to `meta.json`. (Atomic rename ensures readers never see a partial file.)
6. Return a handle storing the directory path and the Meta.

`Finalize(s *Session, exitCode int) error`:
1. Update Meta in place: status=exited, exited_at=now (UTC ms), exit_code=&exitCode.
2. Write `meta.json` atomically (same pattern).

Spec line 76 mandates that meta.json is written **before** the wrapper exits. The wrapper command (Task 18) calls `Finalize` from a `defer` near the top of its handler.

### Task 10: ANSI escape stripping

**Files:**
- Create: `internal/ansi/strip.go`, `internal/ansi/strip_test.go`

- [ ] **Step 1: Write failing test `TestStripANSI`**

Hand-written cases (the spec accepts hand-written tests for unit-level escape handling — golden files in Task 12 cover the full mix). Test cases:

- CSI sequences: `\x1b[31mred\x1b[0m` → `red`. `\x1b[1;33;44m` → empty.
- Multiple in one string, with text in between.
- OSC sequences: `\x1b]0;title\x07` → empty. `\x1b]2;window\x1b\\` → empty (BEL or ST terminator).
- Single-char escapes: `\x1bc` (reset) → empty. `\x1b(B` (charset select) → empty.
- Plain text passes through unchanged: `"hello"` → `"hello"`.
- Truncated escape at end of input: `"foo\x1b[3"` → `"foo"` (drop incomplete sequence — partial fixtures from Task 9 trimming will hit this).
- UTF-8 multi-byte sequences are not corrupted: `"héllo"` → `"héllo"`.

- [ ] **Step 2: Run test, expect FAIL**

- [ ] **Step 3: Implement `Strip(b []byte) []byte`**

Per spec line 138: regex-based or hand-rolled state machine — both fine. Recommended: hand-rolled byte scanner (regex per byte stream is slower and tricky with partial input). State machine handles:
- `\x1b[` → CSI: skip until a final byte in `[0x40-0x7e]`.
- `\x1b]` → OSC: skip until BEL (`\x07`) or ST (`\x1b\\`).
- `\x1b(`, `\x1b)`, `\x1b*`, `\x1b+` → charset select: skip the next byte too.
- `\x1bN`, `\x1bO` (single shift): skip the next byte.
- Other `\x1b<X>` single-char escapes: drop both bytes.
- Truncated at end: drop without panic.

The function operates on byte slices, not strings, to preserve binary input fidelity for non-UTF8 fixtures.

### Task 11: Line discipline (\r-as-redraw)

**Files:**
- Create: `internal/ansi/linedisc.go`, `internal/ansi/linedisc_test.go`

- [ ] **Step 1: Write failing test `TestLineDiscipline`**

Test verifies the behavior described in spec line 139:

- Plain `"hello\nworld\n"` → emits two lines: `"hello"`, `"world"`.
- Bare `\r` overwrites: `"loading...\rloaded"` followed by `\n` → emits `"loaded"` (the bare `\r` reset the pending line; subsequent bytes overwrite).
- Spinner: `"\r| 50%\r| 100%\rdone\n"` → emits `"done"`.
- `\r\n` is treated as line terminator, not redraw: `"line1\r\nline2\n"` → emits `"line1"`, `"line2"`.
- Partial line at end of input (no trailing `\n`): the bytes are buffered and emitted on `Flush()` or on `Close()`. (`Flush` is for tests; in production the wrapper calls `Close` on the discipline when the child exits — Task 14.)
- Stream input (write 1 byte at a time vs all at once) produces the same emitted lines.

The discipline exposes a `LineDiscipline` type with `Write(b []byte) (int, error)` (io.Writer) and `Close() error`. Internally it strips ANSI (calls Task 10) and tracks the current pending line. Each finalized line is delivered via a callback or output channel — pick whatever is cleanest for downstream consumption (Task 12 wires it).

- [ ] **Step 2: Run test, expect FAIL**

- [ ] **Step 3: Implement line discipline**

State machine over the input bytes (post-ANSI-strip):
- Maintain a `pending []byte` (the current line being built).
- On `\n`: emit `pending`, clear it.
- On `\r` followed by `\n`: same as `\n` (treat `\r\n` as one terminator).
- On bare `\r` (not followed by `\n` in this batch): clear `pending`. The next bytes overwrite.

Edge case: `\r` at end of buffer but next batch starts with `\n`. Handle by carrying a "saw_cr" flag across `Write` calls.

The downstream consumer (the file writer in Task 12) is wired via a constructor parameter — e.g. `NewLineDiscipline(emit func(line []byte))`. Test uses a slice-collecting closure.

- [ ] **Step 4: Run test, expect PASS**

- [ ] **Step 5: Commit**

### Task 39: Per-cwd ring buffer eviction at wrapper start

**Files:**

- Modify: `internal/store/session.go` (call eviction in `Create`)

- [ ] **Step 1: Write failing tests`**

`TestRingBufferEviction`:

- Call `EvictOldest(cwd)`.
- Assert: oldest 2 are removed, 2 newest remain on disk.
- Sessions in other cwds are untouched.

`TestRingBufferIdempotent`:
- Call `EvictOldest` twice in a row.
- Second call no-ops (no error, no further removals).

`TestRingBufferConcurrent`:
- Spawn 5 goroutines, each calling `Create(cwd, ...)` simultaneously in the same cwd.
- Wait for all to finish.
- Assert: at most 4 sessions remain (worst case from spec line 185), and no panics. Final state has between 2 and 4 sessions; exact count is best-effort.

`TestRingBufferTolerantOfMissingMeta`:
- Seed 3 sessions in a cwd, delete one's meta.json before calling `EvictOldest`.
- Eviction proceeds without erroring — the session with missing meta is not counted.

- [ ] **Step 2: Run tests, expect FAIL**

- [ ] **Step 3: Implement `EvictOldest(cwd string) error`**

Per spec lines 174–189:
1. Scan all sessions (`store.Scan` already does ENOENT-tolerant reads).
2. Filter to those with matching cwd (exact match — eviction is per-cwd, not ancestor/descendant).
3. Sort by ULID ascending (oldest first).
4. If count < 3: return nil (nothing to evict).
5. Determine oldest entries to remove: indices 0..count-3 (so that count-3+1 = 2 remain after, with the new session about to be added making 3).

Wait — spec line 180 says "If count ≥ 3, os.RemoveAll on the oldest entries until count = 2." Re-read carefully. The eviction is called *before* adding the new one, so the goal is to bring count down to 2 (one slot for the new session).

So: if count ≥ 3, remove oldest until count == 2.

6. For each oldest dir, `os.RemoveAll`. Ignore `ENOENT`. On other errors: log to stderr, continue (best-effort per spec line 189).

### Task 43: `install.sh`

**Files:**
- Create: `install.sh`

- [ ] **Step 1: Write the install script**

Modeled on bun's installer (`https://bun.sh/install`). Per spec lines 399–407:

Behavior:
1. Detect OS and arch via `uname`. Map to GoReleaser asset names (e.g. `darwin-arm64`).
2. If Windows is detected (Cygwin/MSYS uname says `MINGW*` or `MSYS*`): print "see https://peek.sh/install#windows for Scoop install" and exit 1. Do not attempt to install.
3. Determine target directory: `~/.local/bin`. Create if missing.
4. Fetch latest release URL from `https://api.github.com/repos/peek-cli/peek/releases/latest` (or hard-code a version env var).
5. Download the binary archive over HTTPS. No checksum verification in v1 (spec line 405).
6. Extract `peek` into `~/.local/bin/peek`. Chmod +x.
7. Detect whether `~/.local/bin` is on `PATH`. If not, print a helpful message with shell-specific advice:
   - Bash: `echo 'export PATH=$HOME/.local/bin:$PATH' >> ~/.bashrc`
   - Zsh: `~/.zshrc`
   - Fish: `~/.config/fish/config.fish`
   The detection: `case ":$PATH:" in *":$HOME/.local/bin:"*) ;; *) print message ;; esac`.
8. On any download/extraction failure: print clear error pointing at `https://github.com/peek-cli/peek/releases` for manual download, exit 1.

The script is POSIX shell (sh-compatible, not bash-only). No external dependencies beyond `curl`/`wget`, `tar`, `mkdir`, `chmod`, `uname`.
